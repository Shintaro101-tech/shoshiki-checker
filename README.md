# 研究報告書 書式チェッカー（長期研修員用）

長期研修員が作成する研究報告書（.docx／.pdf）の書式を、所内共通の規程に照らして自動チェックする静的Webアプリです。

## 構成

```
.
├── index.html       # メインHTML
├── styles.css       # スタイル
├── app.js           # UI 制御
├── checker.js       # .docx チェックロジック
└── pdf-checker.js   # .pdf チェックロジック
```

外部ライブラリは CDN から読み込み（JSZip、pdf.js）。ビルド不要、Node.js 環境不要です。

## ローカルでの動作確認

ブラウザで `index.html` を開くだけで動作します。Mac/Windows のどちらでも：

- ダブルクリックで開く、または
- ブラウザに index.html をドラッグ＆ドロップ

## 公開URLへのデプロイ

### 推奨: Netlify Drop（最も簡単 / アカウント不要で可）

1. https://app.netlify.com/drop を開く
2. このフォルダ（index.html を含む）を**そのままドラッグ＆ドロップ**
3. 数秒で発行された URL（例: `https://random-name-12345.netlify.app/`）が表示される
4. その URL を所内に配布

カスタムドメインや継続的な管理が必要な場合は無料アカウント登録を推奨。

### 代替: GitHub Pages（Gitに慣れている場合）

1. このフォルダを GitHub のリポジトリにプッシュ
2. リポジトリの Settings → Pages → Source を `main` ブランチに設定
3. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開される

### 代替: Vercel（高速 / 無料）

1. https://vercel.com/new を開く（要アカウント）
2. このフォルダをアップロード、または GitHub と連携
3. 自動的にデプロイされ URL が発行される

## プライバシー

ファイルはすべてブラウザ内で処理され、**サーバには一切送信されません**。研究報告書には未公開の研究内容や個人情報が含まれることがあるため、外部API呼び出しは行っていません。

## チェック項目

仕様書「研究報告書書式設定（所内共通）長研員用 R8 改訂版」に準拠：

- ページ基本設定（余白／字数／行数／字送り／行送り）
- ヘッダー内容（奇数／偶数ページ）
- 教科・領域名／テーマ／サブテーマ／所属・役職・氏名
- 要旨・キーワード（フォント・サイズ・字数・個数）
- 大見出し・小見出し・図表タイトル・本文
- 末尾構造（〈注記〉〈引用文献〉〈生成AI利用〉）
- 引用文献リスト（フォント・括弧・Web資料の参照年月など）
- 禁止事項（太字・下線・特殊文字・全角英数字・行頭スペース など）

各検出項目には、**マニュアル該当箇所（ページ・項番）**と**段落番号・推定行番号**が併記されます。

## .docx と .pdf の差

| 項目 | .docx | .pdf |
|---|---|---|
| ページ余白 | ✅ 設定値で取得 | △ 推定のみ |
| 字送り・行送り | ✅ 設定値で取得 | ❌ 取得不可 |
| フォント名・サイズ | ✅ スタイル継承を含めて正確 | ✅ 表示時の値 |
| 文字種チェック | ✅ | ✅ |
| 末尾構造 | ✅ | ✅ |
| 推奨 | **こちらを推奨** | 補助的 |
